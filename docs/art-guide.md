# 画像素材ガイド — 自作画像をゲームに入れる方法

今のゲームの絵は、ぜんぶプログラムで描いた図形(いわば仮の絵)。
これを「あじさんが用意した画像」に差し替えていくためのガイド。

## 進め方(全体の流れ)

1. **画風をひとつ決める**(いちばん大事。混ぜると安っぽくなる)
   - 例: ドット絵(レトロ・リネ1に近い) / 手描きイラスト風 / 水彩の和風
2. **テストとしてキャラ1体だけ作る** → チャットに貼るか `assets/` フォルダに保存
3. Claudeが組み込んで公開 → 実際のゲーム画面で確認
4. しっくりきたら同じ画風で量産(モンスター→建物→地面の順がおすすめ)

画像を渡すときは「これをプレイヤーにして」のように言うだけでOK。
組み込み作業(コード側)は全部Claudeがやる。

## 必要な画像の仕様

すべて **背景透過のPNG**(透過が難しければ単色背景でOK、Claude側で透過処理する)。

### キャラクター・モンスター

- 1コマ **64×64px**(大型ボスは128×128px)
- 足元が画像の**下端中央**に来るように
- 最小構成: 「立ち姿1枚」だけでも組み込める(左右反転はコード側でやる)
- 理想構成: 歩行アニメ 3コマ × 4方向(=12コマを1枚にまとめた192×256pxのシート)
- まずは立ち姿1枚から始めるのがおすすめ

### 建物・木・岩などの置き物

- 幅128〜256px、高さ自由
- 接地点(地面に着く場所)が下端中央
- 建物は「斜め上から見た角度」で描くとクォータービューに馴染む

### 地面

- 継ぎ目なくつながる(シームレス)テクスチャ **256×256px**
- 草地・土・石畳の3種類から
- ※地面はいちばん難易度が高いので後回しでOK。今のコード描画のままでも成立する

## 画像の作り方の選択肢

1. **AI画像生成**(手軽。おすすめの入り口)
   - コツ: プロンプトに「game asset, isometric view, single character,
     full body, simple background」などを入れる
   - ドット絵なら「pixel art, 64x64, JRPG style」
   - 和風なら「japanese folklore, yokai, muted colors」
   - キャラの向き違い・アニメコマはAIが苦手 → まず立ち姿1枚でよい
2. **自分で描く**(iPadなど。いちばん統一感が出る)
3. **フリー素材**(すぐ試せる。「ぴぽや」等の日本のRPG素材サイトが有名。
   利用規約の確認はClaudeがやるので、使いたい素材のURLを渡すこと)

## 技術メモ(Claude向け)

- 画像導入と同時にプリロード機構を実装し、読み込み失敗時は現行のコード描画に
  フォールバックする(画像が揃うまで混在OKな設計にする)
- 画像を入れた時点でアーティファクト版(単一ファイル)は原則卒業し、
  GitHub Pages URLに一本化(少量ならdata URI埋め込みで延命可)
- スプライトシート対応: 3×4グリッド、1コマ64×64、行順=下向き/左向き/右向き/上向き
