# 画像素材ガイド — 自作画像をゲームに入れる方法

今のゲームの絵は、ぜんぶプログラムで描いた図形(いわば仮の絵)。
これを「あじさんが用意した画像」に差し替えていくためのガイド。

## 進め方(全体の流れ)

1. **画風をひとつ決める**(いちばん大事。混ぜると安っぽくなる)
   - 例: ドット絵(レトロ・リネ1に近い) / 手描きイラスト風 / 水彩の和風
2. **テストとしてキャラ1体だけ作る** → チャットに貼るか `assets/` フォルダに保存
3. Claudeが組み込んで公開 → 実際のゲーム画面で確認
4. しっくりきたら同じ画風で量産(モンスター→建物→地面の順がおすすめ)

画像を渡すときは「これをプレイヤーにして」のように言うだけでOK。
組み込み作業(コード側)は全部Claudeがやる。

## 必要な画像の仕様

すべて **背景透過のPNG**(透過が難しければ単色背景でOK、Claude側で透過処理する)。

### キャラクター・モンスター

- 1コマ **64×64px**(大型ボスは128×128px)
- 足元が画像の**下端中央**に来るように
- 最小構成: 「立ち姿1枚」だけでも組み込める(左右反転はコード側でやる)
- 理想構成: 歩行アニメ 3コマ × 4方向(=12コマを1枚にまとめた192×256pxのシート)
- まずは立ち姿1枚から始めるのがおすすめ

### 建物・木・岩などの置き物

- 幅128〜256px、高さ自由
- 接地点(地面に着く場所)が下端中央
- 建物は「斜め上から見た角度」で描くとクォータービューに馴染む

### ダンジョン共通階段

- `assets/map_stairs_up.png` = 上り階段、`assets/map_stairs_down.png` = 下り階段
- どの洞窟・地下施設にも馴染む無紋の黒灰色石造。640px透過PNGを画面上104〜108pxで描画
- 全ダンジョンの `stairup` / `stairdown` が同じ2素材を使う。読み込み失敗時はCanvas図形へフォールバック

### ダンジョン共通松明

- `assets/map_dungeon_torch_body.png` = 黒鉄の火皿付き本体、`assets/map_dungeon_torch_flame.png` = 炎レイヤー
- 全ダンジョンの `torch` が共通利用。本体は固定し、炎だけを横揺れ・伸縮・明滅させる
- 松明は広い部屋の壁際に1本だけ配置し、細い通路には置かない。経路探索上も通り抜け不可
- 橙色の照明と火の粉も松明ごとに異なる周期で動き、画像を取得できない場合はCanvas描画へフォールバック

### 地面

- 継ぎ目なくつながる(シームレス)テクスチャ **256×256px**
- 草地・土・石畳の3種類から
- ※地面はいちばん難易度が高いので後回しでOK。今のコード描画のままでも成立する

## 画像の作り方の選択肢

1. **AI画像生成**(手軽。おすすめの入り口)
   - コツ: プロンプトに「game asset, isometric view, single character,
     full body, simple background」などを入れる
   - ドット絵なら「pixel art, 64x64, JRPG style」
   - 和風なら「japanese folklore, yokai, muted colors」
   - キャラの向き違い・アニメコマはAIが苦手 → まず立ち姿1枚でよい
2. **自分で描く**(iPadなど。いちばん統一感が出る)
3. **フリー素材**(すぐ試せる。「ぴぽや」等の日本のRPG素材サイトが有名。
   利用規約の確認はClaudeがやるので、使いたい素材のURLを渡すこと)

## 技術メモ(Claude向け)

- 画像導入と同時にプリロード機構を実装し、読み込み失敗時は現行のコード描画に
  フォールバックする(画像が揃うまで混在OKな設計にする)
- 画像を入れた時点でアーティファクト版(単一ファイル)は原則卒業し、
  GitHub Pages URLに一本化(少量ならdata URI埋め込みで延命可)

### 実装済み(v0.4〜v0.4.1)

- プレイヤー: `assets/char_player_m.png` / `char_player_f.png` = 128×128 ×4コマの横並びシート。
  コマ順 = **下向き / 上向き / 左向き / 右向き**(`DIR_FRAME`参照)。
  画面上は48pxで描画。`e.dir`はアイソメ変換後の見た目方向で判定。
  タイトル画面で男女選択(`player.gender`、セーブに保存)
- 武器アイコン: `assets/icon_{tanto,katana,greatsword,bow}.png` = 64×64、45度回転済み。
  `ITEMS[].icon` に指定すると店・装備画面に表示される(弓は未使用の予備)。
  近接攻撃時は装備中の武器アイコンが振り下ろされ、斬撃の軌跡が出る(drawPlayer内)
- 防具・薬アイコン: `assets/icon_armor_{cloth,leather,chain,plate}.png` / `icon_potion.png` =
  Claudeがコードで描いた16×16ドット絵の4倍拡大(仮アイコン)。
  同じ画風のAI生成画像をもらえたら差し替え推奨
- 元画像の加工手順(緑背景クロマキー or ニセ透過の除去→分割→シート化)は
  会話でChatGPT生成画像を渡せばClaudeが再現できる
- 今後同様に `mob_*.png` を渡されたら drawGoblin 等を同方式で差し替える

### 実装済み(v0.5.2) — アニメーションシート

- `char_player_m_walk.png` = 512×512。4行(下/上/左/右)×4コマ(踏み出し→揃う→逆足→揃う)。
  右向きは左向きの反転を焼き込み済み。コマ送りは `e.walkT`(移動距離)に同期
- `char_player_m_attack.png` = 480×640。**セル160px**(頭上の刀が128に収まらないため)。
  4行(下/上=左流用/左/右反転)×3コマ。ヒット判定はアニメ開始0.3秒後(update内)
- `char_player_m_hit.png` / `char_player_m_death.png` = 128×128 各1コマ
- 元画像は緑背景(クロマキー)で受領→greenness判定で透過化。
  スケールは既存立ち絵のコマ内身長(117px)・接地位置(124px)に合わせる
- 女性キャラは未対応(立ち絵のみ)。同じ7枚セットをもらえたら同手順で追加
- 旧 `char_player_walk_*.png`(8コマ白背景方式)は廃止済み

### 輪廻大陸の新素材パイプライン(v0.29以降)

1. 必ず1ファイル1対象で生成し、上下左右に十分な余白を持たせる
2. 竹や樹木など緑の対象は均一な`#ff00ff`背景で生成する
3. imagegenスキルの`remove_chroma_key.py --auto-key border --soft-matte --despill`で個別PNGへ透過化する
4. `python3 scripts/normalize_rinne_asset.py --input <asset.png> --output <asset.png>`で対象の周囲を均等64px透明余白に正規化する
5. `python3 scripts/validate_rinne_asset.py <asset.png>`で四辺透過・48px以上の余白・半透過輪郭・キー色残留を検査する
6. 個別PNGを実画面の草地と石畳、昼と夜で確認し、合格後にのみ本編へ配置する

第一段階の検品合格素材は`rinne_bamboo_pilot.png`、`rinne_machiya_pilot.png`、
`rinne_pine_pilot.png`、`rinne_ginkgo_pilot.png`、`rinne_maple_pilot.png`。
旧アトラスで木の根元に残っていた灰白色の地面縁は、3種とも個別透過PNG化で解消済み。
ただしv0.30で大陸を320×320へ再拡張したため、密度と導線の再設計まで本編では読み込まず、配置もしない。
- 地面は320×320の単一座標系。南岸・街道・都の間にロード画面は入れない
- 北は日本海・若狭湾、中央東寄りは内陸の琵琶湖、南西は大阪湾、東は伊勢湾、南東は志摩半島、南は紀伊半島として地形を分ける
- 関西地形版では橋セルを実マップに配置せず、都=銀杏・楓、街道=竹、紀伊内陸=楠、沿岸・山稜=黒松に植生を分ける
- 当たり判定はマスのまま、地面テクスチャは隣接マスを不規則な連続マスクで塗る。
  ひし形の輪郭線を描かず、道・砂浜・水辺の境界を揺らしてマス目感を消す
- 夜用画像は持たず、南岸・街道・都へリアルタイムの色調・月光を合成する。
  都では灯籠の暖色光も重ねる
- 画像生成時に市松模様が実画素として焼き込まれた場合は
  `scripts/remove_checker_alpha.cjs` で外周と連結した無彩色背景だけを透過する
- 構図検討用の南岸・街道・都の一枚絵は `_wip_src/rinne_continent_concepts/` に保管し、
  配信素材には使わない
