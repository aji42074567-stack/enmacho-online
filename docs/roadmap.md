## v1.0 構想 — マルチプレイ(本物のリネに近づく)

「同じ世界に他のプレイヤーがいる」を実現するにはサーバーが必要。

### 仕組みの説明(かんたん版)

- 今のゲームは各自のブラウザの中だけで動いている
- マルチプレイでは「世界の本体」をサーバーに置き、
  各プレイヤーのブラウザは操作を送信し、世界の状態を受信して描画する
- 通信は WebSocket(常時接続)を使う

### 現実的な構成案

1. プレイヤーの在席・位置・チャットはSupabase Realtimeを利用
2. モンスターや戦闘結果を全員で共有する段階では、サーバー権威型へ移行
   - Cloudflare Workers + Durable Objectsを第一候補とする
   - Durable Objectが「1つのマップ=1つの部屋」を担当する
3. Node.js + Socket.ioをVPSで動かす案は、自由度が必要になった場合の候補

### ステップ分解

1. まず「同じマップに他人が見えて歩ける」だけを作る(チャットなし・戦闘なし) ✅
2. チャット追加
3. モンスターとの戦闘をサーバー側で処理(チート防止)
4. セーブをサーバー側に移す(アカウント制)

一気に作らず、1から順に。1ができた時点でかなり感動があるはず。
